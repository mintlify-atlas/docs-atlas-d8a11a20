Grype is a command-line vulnerability scanner built by [Anchore](https://anchore.com). It catalogs all packages in your target — container images, local directories, or SBOMs — then matches them against a curated vulnerability database to surface known CVEs and advisories.

## Supported image formats

Grype supports all major container image formats:

| Format | Example |
|---|---|
| Docker daemon | `grype yourrepo/yourimage:tag` |
| OCI archive | `grype oci-archive:path/to/yourimage.tar` |
| OCI directory | `grype oci-dir:path/to/yourimage` |
| Podman daemon | `grype podman:yourrepo/yourimage:tag` |
| Singularity (SIF) | `grype singularity:path/to/yourimage.sif` |
| Registry (no daemon) | `grype registry:yourrepo/yourimage:tag` |

## How Grype works

When you run a scan, Grype performs three steps in parallel:

1. **Fetches the vulnerability database.** Grype downloads and caches a vulnerability database aggregated from NVD, GitHub Advisory Database, and OS-specific sources. The database updates automatically on each run.

2. **Catalogs packages.** Grype uses [Syft](https://github.com/anchore/syft) under the hood to enumerate all packages in the target — OS packages (rpm, deb, apk), language packages (pip, npm, gem, jar, etc.), and more.

3. **Matches packages against the database.** Each cataloged package is compared against the vulnerability database using ecosystem-specific matchers (dpkg, rpm, Java, Python, Go, and others). Matches are deduplicated, filtered by any ignore rules, and then displayed.

<Note>
  Grype can also accept an SBOM as input, skipping the cataloging step entirely for faster scans on targets you've already inventoried with Syft.
</Note>

## Supported scan targets

Beyond container images, Grype can scan:

- **Filesystems and directories** — `grype dir:./my-project`
- **Individual files** — `grype file:path/to/binary`
- **SBOMs** — `grype sbom:./sbom.json` (Syft JSON, CycloneDX, SPDX)
- **Package URLs (PURLs)** — `grype purl:path/to/purl/file` or a single PURL directly
- **CPEs** — `grype cpes:path/to/cpes/file` or a single CPE directly (e.g. `cpe:2.3:a:openssl:openssl:3.0.14:*:*:*:*:*:*:*`)
- **Piped SBOM input** — `syft alpine:latest -o json | grype`
