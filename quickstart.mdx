This guide walks you through installing Grype, running your first scan, understanding the output, and using key flags to integrate Grype into your workflow.

## Run your first scan

<Steps>
  <Step title="Install Grype">
    Install the latest release with the install script:

    ```bash
    curl -sSfL https://get.anchore.io/grype | sudo sh -s -- -b /usr/local/bin
    ```

    Confirm the installation:

    ```bash
    grype version
    ```

    See [Installation](/installation) for Homebrew, Docker, Chocolatey, and other options.
  </Step>

  <Step title="Scan a container image">
    Point Grype at any container image by name. Grype pulls it from the Docker daemon (or a registry if not found locally):

    ```bash
    grype alpine:latest
    ```

    Grype will:
    1. Download and cache the vulnerability database (first run only).
    2. Catalog all packages in the image using Syft.
    3. Match packages against the database.
    4. Print a results table to stdout.
  </Step>

  <Step title="Read the output">
    Grype prints a table with one row per vulnerability match:

    ```
    NAME          INSTALLED  FIXED-IN  TYPE  VULNERABILITY   SEVERITY
    libssl3       3.1.4-r2   3.1.4-r5  apk   CVE-2024-0727   Medium
    busybox       1.36.1-r2  1.36.1-r5 apk   CVE-2023-42363  Medium
    ssl_client    1.36.1-r2  1.36.1-r5 apk   CVE-2023-42363  Medium
    ```

    | Column | Description |
    |---|---|
    | `NAME` | Package name affected by the vulnerability |
    | `INSTALLED` | Version currently installed in the scanned target |
    | `FIXED-IN` | Version that contains the fix (`(none)` if no fix is available) |
    | `TYPE` | Package type (e.g., `apk`, `deb`, `rpm`, `python`, `java-archive`) |
    | `VULNERABILITY` | CVE or advisory ID |
    | `SEVERITY` | Severity rating: `Negligible`, `Low`, `Medium`, `High`, or `Critical` |
  </Step>

  <Step title="Scan more target types">
    Grype accepts many input types beyond image names:

    ```bash
    # Scan Ubuntu 20.04
    grype ubuntu:20.04

    # Scan a local directory
    grype dir:./my-project

    # Scan an SBOM file
    grype sbom:./sbom.json

    # Pipe a Syft SBOM into Grype
    syft alpine:latest -o json | grype
    ```

    <Tip>
      Scanning an SBOM with `grype sbom:` or piping from Syft skips the cataloging step, making it the fastest way to scan targets you've already inventoried.
    </Tip>
  </Step>

  <Step title="Fail on high-severity findings">
    Use `--fail-on` (short flag: `-f`) to exit with code `2` when Grype finds vulnerabilities at or above a given severity. This is the standard way to block CI/CD pipelines:

    ```bash
    grype alpine:latest --fail-on high
    ```

    Valid severity values are: `negligible`, `low`, `medium`, `high`, `critical`.

    ```bash
    # Fail on critical only
    grype ubuntu:20.04 --fail-on critical

    # Fail on medium and above
    grype dir:./my-project --fail-on medium
    ```

    <Note>
      Exit code `0` means no vulnerabilities at or above the threshold were found. Exit code `2` means the threshold was exceeded. Exit code `1` means an error occurred.
    </Note>
  </Step>
</Steps>

## Output formats

By default Grype prints a human-readable table. Use `-o` to select a different format:

```bash
# JSON output (machine-readable)
grype alpine:latest -o json

# CycloneDX JSON (for SBOM tooling)
grype alpine:latest -o cyclonedx-json

# SARIF (for GitHub Code Scanning and IDE integrations)
grype alpine:latest -o sarif

# Write output to a file instead of stdout
grype alpine:latest -o json --file results.json
```

Available formats: `table`, `json`, `cyclonedx`, `cyclonedx-json`, `sarif`, `template`.

## Common flags

| Flag | Description |
|---|---|
| `-o FORMAT` | Output format (default: `table`) |
| `-f SEVERITY` | Exit with code 2 if any match meets or exceeds this severity |
| `--only-fixed` | Show only vulnerabilities that have a fix available |
| `--by-cve` | Group results by CVE ID instead of advisory ID |
| `--show-suppressed` | Include ignored matches in table output |
| `--platform PLATFORM` | Override the target platform (e.g., `linux/arm64`) |
| `-s SCOPE` | Layer scope: `squashed` (default) or `all-layers` |
